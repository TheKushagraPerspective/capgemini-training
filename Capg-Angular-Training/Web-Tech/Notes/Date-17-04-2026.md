# Angular Project Structure (Detailed)

---

## 1. Overview

When you create an Angular project using Angular CLI, it generates a structured folder system.

This structure helps in:

* Organizing code
* Managing dependencies
* Building scalable applications

---

## 2. Root Level Files and Folders

---

### 1. package.json

* Contains project metadata
* Lists dependencies and scripts

Key sections:

* dependencies → required for production
* devDependencies → required for development
* scripts → commands like start, build

---

### 2. package-lock.json

* Locks exact versions of installed packages
* Ensures same setup on all systems

---

### 3. node_modules

* Contains all installed packages
* Automatically generated
* Should not be modified manually

---

### 4. tsconfig.json

* Main TypeScript configuration file
* Defines compiler options

Examples:

* target (ES version)
* module system
* strict type checking

---

### 5. tsconfig.app.json

* TypeScript config specific to application code
* Extends base tsconfig.json

---

### 6. tsconfig.spec.json

* Used for testing configuration
* Applied to spec (test) files

---

### 7. angular.json

Main configuration file of Angular project.

Contains:

* Project structure
* Build configuration
* Assets (images, icons, etc.)
* Entry files
* Environment settings

---

### 8. public folder

* Stores static assets
* Example: images, icons, fonts

---

## 3. src Folder (Main Application Code)

This is the most important folder.

---

### index.html

* Main HTML file
* Loaded by browser
* Angular injects app into this file

---

### main.ts

* Entry point of Angular application
* Bootstraps the root module or component

Example:

```ts id="a1"
bootstrapApplication(AppComponent);
```

---

### styles.css

* Global CSS file
* Applies styles across entire application

---

### app Folder

Contains main application logic.

---

## 4. app Folder Structure

---

### app.component.ts

* Main component logic
* Handles data and functionality

---

### app.component.html

* Template (UI structure)
* Defines how data is displayed

---

### app.component.css

* Styles for this component only
* Scoped CSS

---

### app.config.ts

* Configuration for application
* Used in standalone Angular setup

---

### app.config.spec.ts

* Test file for app configuration

---

## 5. Important Angular Concepts

---

### Component

* Basic building block of Angular
* Combines:

  * HTML (view)
  * CSS (style)
  * TypeScript (logic)

---

### Module (older Angular)

* Groups components and services
* Example: AppModule

(Note: New Angular uses standalone components)

---

### Bootstrap

* Process of starting the app
* Done in main.ts

---

### Dependency Injection

* Angular provides services automatically
* Improves code reusability

---

### Assets

Defined in angular.json

Used for:

* Images
* Icons
* Static files

---

## 6. Development vs Production

### Development

* Uses devDependencies
* Faster build, more debugging

---

### Production

* Optimized code
* Smaller bundle size
* Better performance

---

```text id="a2"
project/

├── node_modules/

```

---

## 8. Summary

* Angular projects follow a structured folder system
* src folder contains main application code
* app folder contains components
* main.ts is entry point
* angular.json controls project configuration
* package.json manages dependencies
* node_modules stores installed libraries

This structure helps in building scalable and maintainable applications.

# Angular Application Flow (After `ng serve`)

---

## 1. What happens when you run `ng serve`

```bash
ng serve
```

This command starts a **development server** and builds your Angular application.

---

## 2. High-Level Flow

1. Angular CLI starts dev server
2. Build process starts (esbuild / vite)
3. Files are bundled
4. `index.html` is served to browser
5. Browser loads `main.ts`
6. Angular bootstraps the application
7. App renders on screen

---

## 3. Step-by-Step Detailed Flow

---

## Step 1: Angular CLI starts Dev Server

* Command: `ng serve`
* Starts a local server (usually on `http://localhost:4200`)
* Watches file changes (auto reload)

Purpose:

* Makes development faster
* No need to refresh manually

---

## Step 2: Build Process (Bundling)

Angular uses a **module bundler** like:

* esbuild (modern Angular)
* vite (newer setups)

### What bundler does:

* Combines all files:

  * TypeScript
  * CSS
  * HTML
* Converts TypeScript → JavaScript
* Optimizes code

Output:

* A bundle (single or multiple JS files)

---

## Step 3: Dev Server Injects Files

Dev server:

* Injects all CSS
* Injects bundled JS files
* Serves everything to browser

---

## Step 4: index.html is Loaded

```html
<!-- index.html -->
<body>
  <app-root></app-root>
</body>
```

* This is the main HTML file
* Browser loads this first
* `<app-root>` is just a placeholder

---

## Step 5: Entry Point (main.ts)

```ts
// main.ts
import { bootstrapApplication } from '@angular/platform-browser';
import { AppComponent } from './app/app.component';

bootstrapApplication(AppComponent);
```

### What happens here:

* This is the entry point of Angular
* Angular starts execution from here
* Calls bootstrap function

---

## Step 6: Bootstrapping Process

Bootstrapping means:
Starting the Angular application

### What bootstrap does:

* Loads root component (`AppComponent`)
* Applies configurations (from app.config.ts)
* Initializes dependency injection

---

## Step 7: AppComponent Loads

```ts
@Component({
  selector: 'app-root',
  templateUrl: './app.component.html'
})
export class AppComponent {}
```

* selector = `app-root`
* Angular replaces `<app-root>` in index.html

---

## Step 8: Rendering on UI

* Angular renders HTML from `app.component.html`
* Applies styles from `app.component.css`
* Executes logic from `app.component.ts`

Now UI is visible in browser.

---

## 4. Complete Flow Summary

```text
ng serve
   ↓
Dev Server (localhost:4200)
   ↓
Bundler (esbuild / vite)
   ↓
Bundle JS + CSS
   ↓
index.html loaded
   ↓
main.ts executed
   ↓
bootstrapApplication()
   ↓
AppComponent loaded
   ↓
<app-root> replaced
   ↓
UI Rendered in Browser
```

---

## 5. Important Concepts

---

### 1. Dev Server

* Runs locally
* Auto reloads on changes
* Only for development

---

### 2. Bundling

* Combines multiple files into one
* Improves performance
* Required for browser compatibility

---

### 3. Entry Point

* `main.ts` is entry file
* Execution starts from here

---

### 4. Bootstrapping

* Initializes Angular app
* Loads root component

---

### 5. Component Rendering

* Angular finds selector
* Replaces HTML element
* Displays UI

---

## 6. Role of Each File

| File               | Role           |
| ------------------ | -------------- |
| index.html         | Base HTML page |
| main.ts            | Entry point    |
| app.component.ts   | Logic          |
| app.component.html | UI             |
| app.component.css  | Styling        |
| app.config.ts      | Configuration  |

---

## 7. Development vs Production Flow

### Development (`ng serve`)

* Fast build
* Live reload
* Debugging enabled

### Production (`ng build`)

* Optimized code
* Minified files
* Better performance

---

## 8. Key Interview Points

* Angular starts from `main.ts`
* Bootstrapping loads root component
* Bundler converts TS → JS
* index.html contains root selector
* Dev server handles live reload

---

## 9. Final Understanding

* Angular app does not directly run HTML
* It starts from JavaScript (main.ts)
* Components control UI
* Bootstrapping connects everything

This is the complete flow of Angular after running `ng serve`.
