# Angular Project Setup and Overview

## 1. Project Setup (TypeScript Environment)

Before starting with Angular, we first set up a basic TypeScript environment.

### Steps

```bash
# Step 1: Navigate to root directory
cd root-directory

# Step 2: Initialize Node project
npm init -y

# Step 3: Install TypeScript
npm install typescript --save-dev

# Step 4: Initialize TypeScript config
npx tsc --init
```

### Output

* package.json → stores project configuration
* tsconfig.json → stores TypeScript configuration

---

## 2. Evolution of Angular

### AngularJS (2009)

* Developed by Google
* Created by Misko Hevery and Adam Abrons
* Based on JavaScript
* Used MVC (Model View Controller) architecture

#### Features

* Two-way data binding
* Dependency injection
* Directives

#### Limitations

* Poor performance for large applications
* Difficult to scale
* Complex debugging

---

### Rise of React (2011–2013)

* Developed by Facebook
* Introduced component-based architecture
* Uses Virtual DOM for faster performance

AngularJS started losing popularity because React was faster and simpler.

---

### TypeScript (2013)

* Developed by Microsoft
* Superset of JavaScript

#### Advantages

* Static typing
* Better tooling
* Easier to maintain large codebases

This became the base for modern Angular.

---

### Angular 2+ (2015)

* Complete rewrite by Google
* Built using TypeScript
* Shift from MVC to component-based architecture

---

## 3. Core Concepts of Angular

### Components

Each part of the UI is divided into components.

```ts
@Component({
  selector: 'app-root',
  templateUrl: './app.component.html'
})
export class AppComponent {}
```

---

### Modules

Angular applications are modular.

```ts
@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  bootstrap: [AppComponent]
})
export class AppModule {}
```

---

### Data Binding

* Interpolation → {{ data }}
* Property Binding → [value]="data"
* Event Binding → (click)="handleClick()"
* Two-way Binding → [(ngModel)]="data"

---

### Dependency Injection

```ts
constructor(private service: DataService) {}
```

---

## 4. Angular vs React

| Feature      | Angular         | React            |
| ------------ | --------------- | ---------------- |
| Type         | Framework       | Library          |
| Language     | TypeScript      | JavaScript (JSX) |
| Architecture | Component-based | Component-based  |
| Learning     | Harder          | Easier           |
| Performance  | Good            | Very Fast        |
| Use Case     | Large apps      | Fast UI apps     |

---

## 5. When to Use Angular

* Large scale applications
* Enterprise projects
* Applications needing structure and security

---

## 6. When to Use React

* Fast development
* Dynamic user interfaces
* Lightweight applications

---

## 7. Summary

* AngularJS (2009) → JavaScript + MVC
* React (2011–2013) → Component-based
* TypeScript (2013) → Type safety
* Angular 2+ (2015) → TypeScript + Components

Angular is best for structured and large applications.
React is best for speed and flexibility.

---

## 8. Final Note

Angular is a complete framework with built-in features like routing, forms, and HTTP handling.

React is a library and requires additional packages for full functionality.
