# DOM (Document Object Model)

## 1. What is DOM

DOM (Document Object Model) represents the HTML document as a tree structure.
It allows JavaScript to access and manipulate HTML elements.

---

## 2. DOM Selection Methods (Indirect Access)

These methods are used to select elements from the DOM.

---

### 1. getElementById("id")

* Selects a single element using its id
* Returns: Element or null

#### Important Point

If the id does not exist, it returns null.

#### Example

```js id="a1"
let element = document.getElementById("heading");
```

---

### 2. getElementsByClassName("class")

* Selects elements using class name
* Returns: HTMLCollection

#### Important Point

* If class does not exist → returns empty HTMLCollection
* HTMLCollection is not a real array

To use array methods:

```js id="a2"
let items = document.getElementsByClassName("box");
let arr = Array.from(items);
arr.forEach(el => console.log(el));
```

---

### 3. getElementsByTagName("tag")

* Selects elements using tag name
* Returns: HTMLCollection

#### Important Point

* If tag does not exist → returns empty HTMLCollection
* Needs conversion to array for advanced methods

```js id="a3"
let divs = document.getElementsByTagName("div");
```

---

### 4. querySelector("selector")

* Selects the first matching element
* Returns: Element or null

#### Selector Rules

* id → "#id"
* class → ".class"
* tag → "tag"

#### Example

```js id="a4"
let el = document.querySelector("#heading");
```

---

### 5. querySelectorAll("selector")

* Selects all matching elements
* Returns: NodeList

#### Important Point

* If no match → returns empty NodeList
* NodeList supports forEach directly

```js id="a5"
let items = document.querySelectorAll(".box");
items.forEach(el => console.log(el));
```

---

## 3. HTMLCollection vs NodeList

| Feature          | HTMLCollection          | NodeList                      |
| ---------------- | ----------------------- | ----------------------------- |
| Type             | Collection of elements  | Collection of nodes           |
| forEach()        | Not available directly  | Available directly            |
| Convert to Array | Required (Array.from()) | Not required                  |
| Nature           | Live (auto-updates)     | Static (does not auto-update) |

---

## 4. Live vs Static

### HTMLCollection (Live)

* Automatically updates when DOM changes

Example:
If you add a new element, it appears in collection automatically.

---

### NodeList (Static)

* Does not update automatically
* Need to re-select elements if DOM changes

---

## 5. Direct vs Indirect Access

### Indirect Access

* Using selection methods (like above)

### Direct Access

* Using document properties

```js id="a6"
document.body
document.title
document.links
```

---

## 6. Best Practice

* Use querySelector for single element
* Use querySelectorAll for multiple elements
* Avoid old methods unless needed
* Prefer NodeList over HTMLCollection for flexibility

---

## 7. Common Mistakes

* Forgetting "#" for id in querySelector
* Trying to use forEach on HTMLCollection
* Not checking for null before using element

---

## 8. Summary

* DOM allows JavaScript to interact with HTML
* getElementById returns single element or null
* getElementsByClassName and getElementsByTagName return HTMLCollection
* querySelector returns first match
* querySelectorAll returns NodeList
* HTMLCollection is live, NodeList is static
