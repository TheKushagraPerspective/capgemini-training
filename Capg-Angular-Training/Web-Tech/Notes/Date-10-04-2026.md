# JavaScript Objects, JSON, and Asynchronous JavaScript

## 1. Objects in JavaScript

An object represents a real-world entity.
It stores data in key-value pairs.

### Example

```js id="o1"
let user = {
    name: "Kushagra",
    age: 22,
    greet: function() {
        console.log("Hello");
    }
};
```

---

## 2. Object Properties and Methods

### 1. keys()

Returns all keys of an object.

```js id="o2"
Object.keys(user);
```

---

### 2. values()

Returns all values of an object.

```js id="o3"
Object.values(user);
```

---

### 3. entries()

Returns key-value pairs in array format.

```js id="o4"
Object.entries(user);
```

---

### 4. seal()

* Cannot add or delete properties
* Can modify existing properties

```js id="o5"
Object.seal(user);
```

---

### 5. isSealed()

Checks whether object is sealed.

```js id="o6"
Object.isSealed(user);
```

---

### 6. freeze()

* Cannot add, delete, or modify properties

```js id="o7"
Object.freeze(user);
```

---

### 7. isFrozen()

Checks whether object is frozen.

```js id="o8"
Object.isFrozen(user);
```

---

## 3. Additional Important Concepts

### Property Access

```js id="o9"
user.name        // dot notation
user["age"]      // bracket notation
```

---

### Adding and Updating Properties

```js id="o10"
user.city = "Punjab";     // add
user.age = 23;            // update
```

---

### Deleting Properties

```js id="o11"
delete user.city;
```

---

### Looping Through Object

```js id="o12"
for (let key in user) {
    console.log(key, user[key]);
}
```

---

## 4. JSON (JavaScript Object Notation)

JSON is a standard format for data exchange.

### Key Points

* Data is stored in key-value format
* Keys and string values must be in double quotes
* Used in APIs and data transfer

---

### Convert Object to JSON

```js id="j1"
let jsonData = JSON.stringify(user);
```

---

### Convert JSON to Object

```js id="j2"
let obj = JSON.parse(jsonData);
```

---

## 5. Asynchronous JavaScript

Asynchronous JavaScript allows code to run without blocking execution.

It is used when tasks take time, like:

* API calls
* Timers
* File operations

---

## 6. Timing Functions

### setTimeout()

Runs a function after a delay (in milliseconds).

```js id="a1"
setTimeout(() => {
    console.log("Hello after 2 seconds");
}, 2000);
```

---

### setInterval()

Runs a function repeatedly after a fixed interval.

```js id="a2"
let interval = setInterval(() => {
    console.log("Running every 1 second");
}, 1000);
```

---

### clearTimeout()

Stops a timeout before execution.

```js id="a3"
let t = setTimeout(() => {
    console.log("Will not run");
}, 2000);

clearTimeout(t);
```

---

### clearInterval()

Stops an interval.

```js id="a4"
clearInterval(interval);
```

---

## 7. Event Loop (Basic Idea)

JavaScript is single-threaded but handles async operations using:

* Call Stack
* Web APIs
* Callback Queue
* Event Loop

The event loop ensures async tasks run at the right time.

---

## 8. Summary

* Objects store data in key-value pairs
* Object methods: keys, values, entries, seal, freeze
* JSON is used for data exchange
* stringify converts object to JSON
* parse converts JSON to object
* setTimeout runs once after delay
* setInterval runs repeatedly
* Async JavaScript helps avoid blocking code
