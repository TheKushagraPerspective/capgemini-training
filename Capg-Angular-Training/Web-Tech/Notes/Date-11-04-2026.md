# JavaScript Promises and Async/Await

## 1. What is a Promise

A Promise in JavaScript is an object that represents the eventual completion or failure of an asynchronous operation and its result.

It acts as a placeholder for a value that is not available immediately but will be available in the future.

In simple words:
A Promise tells whether an async task is completed, failed, or still running.

---

## 2. Why Promises are Used

### Managing Asynchronous Tasks

Used for operations like:

* API calls
* File loading
* Database operations

These tasks take time and should not block the UI.

---

### Avoiding Callback Hell

Promises help avoid deeply nested callbacks and make code more readable.

---

### Better Error Handling

Using `.catch()`, errors from multiple steps can be handled in one place.

---

### Chaining Operations

Using `.then()`, multiple async operations can run in sequence.

---

### Parallel Execution

Using `Promise.all()`, multiple async tasks can run at the same time.

---

## 3. States of a Promise

### Pending

* Initial state
* Operation is still running

### Fulfilled (Resolved)

* Operation completed successfully
* Result is available

### Rejected

* Operation failed
* Error or reason is available

### Settled

* Final state (either fulfilled or rejected)

---

## 4. Creating a Promise

```js id="p1"
let promise = new Promise((resolve, reject) => {
    let success = true;

    if (success) {
        resolve("Task completed");
    } else {
        reject("Task failed");
    }
});
```

---

## 5. Consuming a Promise

### .then()

```js id="p2"
promise.then((result) => {
    console.log(result);
});
```

---

### .catch()

```js id="p3"
promise.catch((error) => {
    console.log(error);
});
```

---

### .finally()

```js id="p4"
promise.finally(() => {
    console.log("Execution finished");
});
```

---

## 6. Promise Chaining

```js id="p5"
promise
    .then((res) => {
        console.log(res);
        return "Next step";
    })
    .then((data) => {
        console.log(data);
    })
    .catch((err) => {
        console.log(err);
    });
```

---

## 7. Promise Utility Methods

### Promise.all()

```js id="p6"
Promise.all([promise1, promise2])
    .then((results) => console.log(results))
    .catch((err) => console.log(err));
```

---

### Promise.race()

```js id="p7"
Promise.race([promise1, promise2])
    .then((res) => console.log(res))
    .catch((err) => console.log(err));
```

---

## 8. Introduction to Async/Await

Async/Await is a modern way to handle asynchronous operations in JavaScript.
It is built on top of Promises and makes code look like synchronous code.

---

## 9. async Function

An `async` function always returns a Promise.

```js id="a1"
async function demo() {
    return "Hello";
}
```

---

## 10. await Keyword

`await` pauses the execution until the Promise is resolved.

```js id="a2"
async function getData() {
    let result = await promise;
    console.log(result);
}
```

---

## 11. Error Handling with Async/Await

Use try...catch for handling errors.

```js id="a3"
async function fetchData() {
    try {
        let result = await promise;
        console.log(result);
    } catch (error) {
        console.log(error);
    }
}
```

---

## 12. Sequential vs Parallel Execution

### Sequential (one after another)

```js id="a4"
let r1 = await promise1;
let r2 = await promise2;
```

---

### Parallel (faster)

```js id="a5"
let results = await Promise.all([promise1, promise2]);
```

---

## 13. Important Notes

* async/await is just a cleaner syntax over Promises
* await can only be used inside async functions
* Always use try...catch for error handling
* async functions always return a Promise

---

## 14. Promise vs Async/Await

| Feature        | Promise (.then)   | Async/Await          |
| -------------- | ----------------- | -------------------- |
| Syntax         | Chain-based       | Looks synchronous    |
| Readability    | Moderate          | Easy and clean       |
| Error Handling | .catch()          | try...catch          |
| Debugging      | Harder            | Easier               |
| Usage          | Good for chaining | Best for simple flow |

---

## 15. Summary

* Promise handles asynchronous operations
* States: pending, fulfilled, rejected
* .then(), .catch(), .finally() are used to handle results
* Async/Await is built on top of Promises
* Makes code cleaner and easier to understand
* Use Promise for chaining and parallel tasks
* Use async/await for readable and structured code
