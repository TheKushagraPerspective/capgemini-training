# JavaScript Basics and Core Concepts

## 1. History of JavaScript

* In 1995, Brendan Eich created a language called Mocha in 10 days

* Later renamed to LiveScript (September 1995)

* Finally renamed to JavaScript (October 1995)

* In 1996, JavaScript was handed over to ECMA (European Computer Manufacturers Association)

* In 1997, ECMAScript (ES1) was released

* ES6 (major standard version) was released in 2015

* Latest versions continue to release yearly (ES2025 and beyond)

---

## 2. What is JavaScript

JavaScript is a high-level, object-based programming language used to create interactive web applications.

---

## 3. Features of JavaScript

* Scripting language
* Synchronous by default (can handle async)
* Interpreted language
* Single-threaded
* Dynamic language
* Loosely typed (weakly typed)
* Object-based
* Client-side scripting language

---

## 4. Advantages of JavaScript

* Strong community support
* Easy to learn and lightweight
* Fast execution

---

## 5. Disadvantages of JavaScript

* Less secure (runs on client side)
* Vulnerable to XSS (Cross-Site Scripting) attacks

---

## 6. Runtime Environment

### 1. Browser Runtime Environment

Ways to use JavaScript in browser:

* Inline JavaScript
* Internal JavaScript
* External JavaScript

---

### 2. Non-Browser Runtime Environment

* Uses Node.js
* Allows JavaScript to run outside browser

---

## 7. Rendering Engine vs JavaScript Engine

| Feature    | Rendering Engine       | JavaScript Engine                |
| ---------- | ---------------------- | -------------------------------- |
| Purpose    | Displays web page      | Executes JavaScript              |
| Input      | HTML, CSS, Images      | JavaScript code                  |
| Output     | DOM + Render Tree + UI | Executed code / DOM manipulation |
| Dependency | Browser dependent      | Can run independently (Node.js)  |

---

## 8. JavaScript Engine (Working)

### Steps:

1. Parser

   * Breaks code into tokens
   * Converts into AST (Abstract Syntax Tree)

2. Interpreter

   * Converts AST into bytecode

3. JIT Compiler (Just-In-Time)

   * Optimizes frequently used code (hot code)
   * Improves performance

## 10. Short Circuiting

Short circuiting is used in logical operators.

### AND (&&)

* Stops if any value is false
* Returns first false value

### OR (||)

* Stops if any value is true
* Returns first true value

```js id="s1"
console.log(false && "Hello"); // false
console.log(true || "Hello");  // true
```

---

## 11. Hoisting

Hoisting means JavaScript moves declarations to the top of the scope before execution.

```js id="h1"
console.log(a); // undefined
var a = 10;
```

---

## 12. Temporal Dead Zone (TDZ)

* Applies to let and const
* Time between variable usage and declaration

```js id="h2"
console.log(a); // error
let a = 10;
```

---

## 13. var vs let vs const

| Feature   | var      | let       | const     |
| --------- | -------- | --------- | --------- |
| Scope     | Function | Block     | Block     |
| Hoisting  | Yes      | Yes (TDZ) | Yes (TDZ) |
| Reassign  | Yes      | Yes       | No        |
| Redeclare | Yes      | No        | No        |

---

## 14. Additional Important Concepts

### Event Loop (Basic)

JavaScript handles async operations using:

* Call Stack
* Web APIs
* Callback Queue
* Event Loop

---

### Execution Context

JavaScript runs code inside execution contexts:

* Global Execution Context
* Function Execution Context

---

### Call Stack

* Keeps track of function execution
* Works in LIFO (Last In First Out)

---

## 15. Summary

* JavaScript was created in 1995 and standardized by ECMA
* It is a high-level, dynamic, single-threaded language
* Runs in browser and Node.js
* Rendering engine displays UI, JS engine executes code
* JIT compiler improves performance
* Short circuiting optimizes logical operations
* Hoisting moves declarations to top
* TDZ restricts access before initialization
* Event loop handles async behavior
