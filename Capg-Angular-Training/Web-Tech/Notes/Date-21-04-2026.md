# Angular Component Communication Notes (Detailed Guide)

---

# 1. Introduction to Component Communication

Angular applications are built using multiple components. Each component is responsible for a specific part of the UI.

In real-world applications, components often need to share data with each other. This process is called **component communication**.

---

# 2. Types of Component Communication

There are two primary types:

1. Parent to Child Communication (Downward flow)
2. Child to Parent Communication (Upward flow)

---

# 3. Parent to Child Communication

## 3.1 Concept

In this type, the parent component sends data to the child component.

This is achieved using:

* `@Input()` decorator
* Property binding `[ ]`

---

## 3.2 How It Works

1. Parent defines data in its TypeScript file.
2. Parent passes data using property binding in HTML.
3. Child receives data using `@Input()`.
4. Child displays or uses the data.

---

## 3.3 Parent Component Code

### parent.ts

```ts
export class Parent {

  parentMessage = "Hello from Parent";

  fruits = ['Apple', 'Banana', 'Mango', 'Kiwi'];

  user = {
    name: 'teju',
    age: 230
  };

}
```

Explanation:

* `parentMessage` is a string.
* `fruits` is an array.
* `user` is an object.
* These variables act as the source of data.

---

### parent.html

```html
<app-child 
  [message]="parentMessage"
  [fruits]="fruits"
  [user]="user">
</app-child>
```

Explanation:

* `[message]`, `[fruits]`, `[user]` are property bindings.
* They send data from parent to child.
* Left side = child property
* Right side = parent variable

---

## 3.4 Child Component Code

### child.ts

```ts
import { Component, Input } from '@angular/core';

export class Child {

  @Input() message!: string;
  @Input() fruits!: string[];
  @Input() user!: { name: string; age: number };

}
```

Explanation:

* `@Input()` allows the child to receive data.
* Each variable corresponds to what parent sends.
* `!` (non-null assertion) tells TypeScript that value will be assigned later.

---

### child.html

```html
<h3>Child Component</h3>

<p>{{ message }}</p>

<h4>Fruits List:</h4>
<ul>
  <li *ngFor="let fruit of fruits">
    {{ fruit }}
  </li>
</ul>

<h4>User Details:</h4>
<p>Name: {{ user.name }}</p>
<p>Age: {{ user.age }}</p>
```

Explanation:

* `{{ message }}` displays string data.
* `*ngFor` loops through array.
* `user.name` and `user.age` access object properties.

---

## 3.5 Key Points

* `@Input()` is required to receive data.
* Property binding `[ ]` is used to send data.
* Supports string, array, object, boolean, etc.
* Data flow is one-way (parent to child).

---

## 3.6 Flow Summary

Parent.ts → Parent.html → Child.ts (@Input) → Child.html

---

# 4. Child to Parent Communication

## 4.1 Concept

In this type, the child component sends data back to the parent component.

This is achieved using:

* `@Output()` decorator
* `EventEmitter`
* Event binding `( )`

---

## 4.2 How It Works

1. Child defines an event using `@Output()`.
2. Child emits data using `EventEmitter`.
3. Parent listens to the event using `(eventName)`.
4. Parent handles data using a function.

---

## 4.3 Child Component Code

### child.ts

```ts
import { Component, Input, Output, EventEmitter } from '@angular/core';

export class Child {

  @Input() message!: string;

  @Output() sendData = new EventEmitter<string[]>();

  sendMessageToParent() {
    this.sendData.emit(['Apple', 'Banana', 'Mango', 'Kiwi']);
  }

}
```

Explanation:

* `@Output()` is used to send data out of the child.
* `EventEmitter<string[]>` defines that an array of strings will be sent.
* `emit()` sends data to the parent.

---

### child.html

```html
<button (click)="sendMessageToParent()">
  Send Data to Parent
</button>
```

Explanation:

* `(click)` is event binding.
* When button is clicked, function is executed.

---

## 4.4 Parent Component Code

### parent.ts

```ts
export class Parent {

  childMessage: string[] = [];

  receiveMessage(data: string[]) {
    this.childMessage = data;
  }

}
```

Explanation:

* `childMessage` stores data received from child.
* `receiveMessage()` handles incoming data.

---

### parent.html

```html
<app-child 
  (sendData)="receiveMessage($event)">
</app-child>

<h4>Received Fruits:</h4>
<ul>
  <li *ngFor="let fruit of childMessage">
    {{ fruit }}
  </li>
</ul>
```

Explanation:

* `(sendData)` listens to child event.
* `$event` contains data sent from child.
* Data is passed to `receiveMessage()`.

---

## 4.5 Key Points

* `@Output()` is required to send data.
* `EventEmitter` is used to emit data.
* `(eventName)` is used in parent to listen.
* `$event` holds emitted data.

---

## 4.6 Flow Summary

Child.ts → emit() → Parent.html → Parent.ts → UI update

---

# 5. Comparison Between Both

| Feature   | Parent to Child | Child to Parent  |
| --------- | --------------- | ---------------- |
| Decorator | @Input()        | @Output()        |
| Syntax    | [property]      | (event)          |
| Direction | Downward        | Upward           |
| Purpose   | Send data       | Send events/data |

---

# 6. Important Concepts

## 6.1 One-Way Data Flow

Angular follows one-way data flow:

* Parent to Child via Input
* Child to Parent via Output (events)

---

## 6.2 Naming Consistency

The names must match exactly:

```ts
@Output() sendData
```

```html
(sendData)="receiveMessage($event)"
```

---

## 6.3 Data Types

Always use correct types:

* string (correct)
* String (incorrect)

---

# 7. Common Mistakes

* Forgetting to use `@Input()`
* Forgetting to use `@Output()`
* Using wrong data types (String instead of string)
* Importing Parent inside Child (incorrect design)
* Not using `$event` in parent

---

# 8. Best Practices

* Keep components loosely coupled
* Avoid direct dependency between parent and child
* Use proper typing
* Reuse data instead of hardcoding

Example:

```ts
this.sendData.emit(this.fruits);
```

---

# 9. Quick Revision

Parent to Child:

* @Input()
* [property]

Child to Parent:

* @Output()
* EventEmitter
* (event)
* $event

---

# 10. Conclusion

Component communication is a fundamental concept in Angular.

Understanding Input and Output allows:

* Better component design
* Reusability
* Clean architecture

These concepts are used in almost every Angular application.

---

# Angular Template Reference Variables (Complete Guide)

---

# 1. Introduction

A **Template Reference Variable** in Angular is a way to directly access an HTML element, Angular component, or directive inside the template.

It is defined using the `#` symbol.

---

# 2. What is a Template Reference Variable?

A template reference variable is a **local reference** to an element in the HTML template.

It allows you to:

* Access input values
* Access DOM elements
* Pass data to functions
* Interact with child components

---

# 3. Syntax

```html
<input #myInput>
```

* `#myInput` is the reference variable
* It refers to this input element

---

# 4. Basic Example

## 4.1 HTML Code

```html
<input #nameInput type="text" placeholder="Enter name">
<button (click)="showValue(nameInput.value)">Show</button>
```

## 4.2 TypeScript Code

```ts
showValue(value: string) {
  console.log(value);
}
```

---

## 4.3 Explanation

* `#nameInput` creates a reference to the input field
* `nameInput.value` accesses the entered value
* The value is passed to the function on button click

---

# 5. Without Template Reference (Traditional Way)

```html
<input [(ngModel)]="name">
<button (click)="showValue()">Show</button>
```

```ts
name: string = "";

showValue() {
  console.log(this.name);
}
```

---

## Difference

| Template Reference       | ngModel              |
| ------------------------ | -------------------- |
| Works directly with DOM  | Uses two-way binding |
| No variable needed in TS | Requires TS variable |
| Simpler for small tasks  | Better for forms     |

---

# 6. Accessing DOM Elements

```html
<input #inputBox type="text">
<button (click)="inputBox.focus()">Focus Input</button>
```

Explanation:

* `inputBox` directly refers to DOM element
* You can call methods like `.focus()`

---

# 7. Passing Reference to Function

```html
<input #numInput type="number">
<button (click)="printValue(numInput.value)">Print</button>
```

```ts
printValue(value: number) {
  console.log(value);
}
```

---

# 8. Using with *ngFor

```html
<ul>
  <li *ngFor="let item of items; let i = index" #listItem>
    {{ item }}
  </li>
</ul>
```

Explanation:

* Reference is created for each iteration
* Useful for DOM manipulation (advanced cases)

---

# 9. Accessing Child Component Using Template Reference

## Parent HTML

```html
<app-child #childComp></app-child>

<button (click)="childComp.someMethod()">
  Call Child Method
</button>
```

## Child TS

```ts
someMethod() {
  console.log("Method called from parent");
}
```

---

## Explanation

* `#childComp` refers to child component instance
* Parent can call methods directly

---

# 10. Key Points

* Defined using `#variableName`
* Works only inside template (HTML)
* Can access:

  * DOM elements
  * Component instances
  * Directive instances

---

# 11. Scope

* Template reference variables are **local**
* Cannot be accessed in TypeScript directly
* Only usable inside HTML template

---

# 12. Common Mistakes

* Trying to use template reference in `.ts` file
* Forgetting `#` symbol
* Confusing with `@ViewChild` (advanced concept)

---

# 13. When to Use

Use template reference variables when:

* You need quick access to input values
* You want to avoid creating unnecessary variables
* You need to call child component methods

---

# 14. Interview / Viva Questions

1. What is a template reference variable?
2. How do you define it?
3. Can it be accessed in TypeScript?
4. Difference between template reference and ngModel?
5. Can we access child component using it?

---

# 15. Quick Revision

* `#var` → creates reference
* Used inside HTML only
* Access using `.value`
* Can call functions and methods

---

# 16. Summary

Template reference variables provide a simple and efficient way to interact with DOM elements and components directly from the template without extra TypeScript code.

---

# Angular Content Projection (ng-content) – Complete Guide

---

# 1. Introduction

Content Projection is a technique in Angular that allows you to **pass HTML content from a parent component to a child component**.

Instead of passing data via TypeScript (`@Input()`), here we pass **actual HTML structure**.

---

# 2. What is Content Projection?

Content Projection means:

> Sending HTML content from parent → child and displaying it inside the child using `<ng-content>`

---

# 3. Why Do We Need It?

Normally:

* Child component has fixed HTML

But sometimes:

* We want reusable components with **dynamic content**

Example:

* Card component
* Modal component
* Layout wrapper

👉 Instead of hardcoding content inside child, we allow parent to inject content

---

# 4. Basic Syntax

## Child Component (Receiver)

```html id="ntdsq9"
<ng-content></ng-content>
```

---

## Parent Component (Sender)

```html id="4n7zbx"
<app-child>
  <p>This content comes from parent</p>
</app-child>
```

---

# 5. How It Works (Step-by-Step)

1. Parent writes content inside `<app-child>`
2. Angular captures that content
3. Child displays it at `<ng-content>` location

---

# 6. Complete Example

---

## 6.1 Child Component

### child.html

```html id="5fztxt"
<h3>Child Component</h3>

<ng-content></ng-content>
```

---

### child.ts

```ts id="kw3k1t"
import { Component } from '@angular/core';

export class Child {}
```

---

## 6.2 Parent Component

### parent.html

```html id="9nqg9t"
<app-child>
  <p>This is projected content from parent</p>
  <button>Click Me</button>
</app-child>
```

---

## 6.3 Output

Child will render:

* Heading from child
* Paragraph + button from parent

---

# 7. Key Difference from @Input()

| Feature      | @Input()        | Content Projection |
| ------------ | --------------- | ------------------ |
| Type of data | JavaScript data | HTML content       |
| Usage        | Data binding    | UI structure       |
| Flexibility  | Limited         | Highly flexible    |

---

# 8. Multiple Content Projection (Advanced)

Angular allows projecting different parts using selectors.

---

## Child HTML

```html id="b8e8sf"
<h3>Header</h3>
<ng-content select="[header]"></ng-content>

<h3>Body</h3>
<ng-content></ng-content>

<h3>Footer</h3>
<ng-content select="[footer]"></ng-content>
```

---

## Parent HTML

```html id="c4qyb7"
<app-child>
  <div header>This is header content</div>

  <p>This is main body content</p>

  <div footer>This is footer content</div>
</app-child>
```

---

## Explanation

* `select="[header]"` → picks element with attribute `header`
* Default `<ng-content>` → gets remaining content
* Allows structured layout

---

# 9. Real-World Example (Card Component)

---

## Child (card.html)

```html id="a2b9zi"
<div class="card">
  <div class="card-header">
    <ng-content select="[card-title]"></ng-content>
  </div>

  <div class="card-body">
    <ng-content></ng-content>
  </div>
</div>
```

---

## Parent

```html id="g1m4pl"
<app-card>
  <h2 card-title>Product Title</h2>

  <p>This is product description</p>
</app-card>
```

---

# 10. Important Points

* `<ng-content>` acts as a placeholder
* It does not create new DOM
* Content is rendered where `<ng-content>` is placed
* Works only in templates (HTML)

---

# 11. Limitations

* Cannot conditionally render `<ng-content>` using `*ngIf`
* Cannot directly manipulate projected content in child TS

---

# 12. Common Mistakes

* Forgetting `<ng-content>`
* Using wrong selectors
* Expecting it to behave like `@Input()`

---

# 13. When to Use

Use content projection when:

* You want reusable UI components
* You want flexible layouts
* You need dynamic HTML content

---

# 14. Interview / Viva Questions

1. What is content projection?
2. Difference between @Input and ng-content?
3. What is `<ng-content>`?
4. Can we have multiple ng-content?

---

# 15. Quick Revision

* Parent sends HTML
* Child receives using `<ng-content>`
* Used for flexible UI
* Supports multiple slots

---

# 16. Summary

Content projection allows Angular components to be more reusable and flexible by enabling dynamic insertion of HTML content from parent components.
