rendering engine vs javascript engine
Key Differences
Feature 	    Rendering Engine (Layout Engine)	                                JavaScript Engine
Primary Goal	Transforms HTML and CSS into a visual representation on the screen.	Parses and executes JavaScript code to make the page interactive.
Inputs	        HTML documents, CSS files, images, and XML.	                        JavaScript code (.js) and WebAssembly.
Core Output	    The Document Object Model (DOM), Render Tree, and visual pixels.	Executed machine code or manipulation of the DOM via APIs.
Independence	Tightly coupled with the browser; rarely used outside of one.	    Can run independently of a browser (e.g., Node.js).

JavaScriptEngine :- 
parser :- first the code will be converted in chunks and then parse the chunks into AST format (Abstract Syntax Tree) tree like structure
then these chunks goes to interpreter then code is converted to bytecode (the code which is neither understandable by humans nor machine)
and JIT(just in time) compiler is also there so after interpreter, the code will go to JIt which is having profiler
this profiler will find the hot code(the code which is repeating again and again like loop/function) and that code will be optimized by the JIT compiler
